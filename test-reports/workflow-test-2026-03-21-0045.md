## Issues Found

### ISSUE-1: `/storage/uploads` Permission Denied (CSV & OCR upload)
- **Severity:** CRITICAL
- **Service:** telemetry-ingestion
- **Step:** 5, 6
- **Title:** File uploads fail with PermissionError on `/storage/uploads`
- **Expected:** CSV/OCR upload returns 202 with job_id
- **Actual:** 500 Internal Error — `PermissionError: [Errno 13] Permission denied: '/storage/uploads'`
- **Root cause:** The `ingestion-storage` Docker volume mounts at `/storage` owned by `root`, but the service runs as user `dialed`. The `/storage/uploads` directory doesn't exist and can't be created.
- **Fix:** Add `RUN mkdir -p /storage/uploads && chown dialed:dialed /storage` to the telemetry-ingestion Dockerfile before `USER dialed`, or add an init container / entrypoint script.

### ISSUE-2: Frontend `VITE_GATEWAY_URL` breaks remote access via tunnel
- **Severity:** CRITICAL
- **Service:** frontend
- **Step:** N/A (tunnel access)
- **Title:** Frontend bakes `http://localhost:8000` as API base URL, breaking remote access
- **Expected:** Frontend API calls should work via `https://dialed.senorai.qzz.io`
- **Actual:** Browser tries to call `http://localhost:8000/api/v1/...` which is unreachable remotely
- **Root cause:** `VITE_GATEWAY_URL=http://localhost:8000` is baked into the JS bundle at build time. The nginx config already has `/api/` → `gateway:8000` proxy, so the frontend should use a relative URL (empty string or just `/`).
- **Fix:** Set `VITE_GATEWAY_URL` to empty string `""` in docker-compose.yml (or remove it). The nginx proxy already handles routing. The `client.ts` already falls back to `''` when the env var is missing.

### ISSUE-3: Voice transcript endpoint requires audio file, no text-only mode via gateway
- **Severity:** HIGH
- **Service:** telemetry-ingestion / gateway
- **Step:** 7
- **Title:** Voice endpoint only accepts audio file upload, no JSON transcript-only path via gateway
- **Expected:** POST `/api/v1/ingest/voice` with JSON `{"session_id": "...", "transcript": "..."}` returns entities
- **Actual:** 422 — `body → file: Field required` (expects multipart file upload)
- **Detail:** The telemetry service has a separate `/ingest/voice/transcript` endpoint that accepts JSON, but the gateway doesn't route to it. The main `/ingest/voice` requires a file upload.
- **Fix:** Either (a) add gateway route for `/api/v1/ingest/voice/transcript` → telemetry `/ingest/voice/transcript`, or (b) make the `file` field optional on the main voice endpoint when `transcript` is provided.
